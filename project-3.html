Project Overview

For my Capstone project, I was hired as a front-end developer at BestBank, which had been lagging behind in terms of website updates. Their last website update was way back in 2005 (AKA the stone ages). My task was to revamp their entire site, and that's where my expertise came into play.

The main objective of this project was to transform the design specifications provided by the designer in Figma into an actual website using my HTML and CSS skills. While the focus was primarily on HTML and CSS, I was given additional challenges to take on so I did it. Firstly, instead of hard coding the account financial data, I had to fetch it from a JavaScript file. Secondly, I had to find a way to display categorized spending for each bank account when clicked on. I not only successfully achieved both stretch goals but also decided to utilize React to build this project in addition to making the layout responsive.

Upon code review, the feedback highlighted an opportunity for improvement: reducing the code length. Although the specific changes were not mentioned, I took it upon myself to carefully examine the codebase for possible optimizations.

One significant change I made was minimizing the number of states used. For instance, in my App component, I started with nine states, but I managed to bring that down to just two states.

I also noticed instances where I unnecessarily repeated myself, largely due to the number of states I had created.

This experience has been valuable in my growth as a developer. While I understood the importance of DRY (Don't Repeat Yourself) code before, I now have a much deeper comprehension and appreciation for it."
